Cap-height ratios are approximations — published per-em values, not values measured at specific point sizes on specific platforms or rendering environments. Numbers will be in the right ballpark, but anyone doing certification work should verify by measuring on the actual target. The hinting quality badge indicates where extra caution is warranted.

px ↔ pt conversion uses the CSS convention 1 pt = 1.333 CSS px (96 ÷ 72). Native desktop applications that work with physical points may differ slightly from this figure.

About this calculator

The calculator answers the question: given a display and a viewing distance, how tall must a character be on screen to remain legible? ISO 9241-303 defines three arc-minute thresholds (16′ / 20′ / 22′) that translate viewing distance into a minimum physical character height — the calculator converts those into dpx and CSS px for your specific screen.

dpx means display pixels — physical pixels on the panel. It is distinct from CSS px, which the browser scales by the device pixel ratio (DPR). High-DPR screens (Retina, modern phones) render 1 CSS px as several dpx.

Apple iPhone Pro / Pro Max

These models use a three-layer rendering pipeline. The browser can only see the first two layers — the hardware pixel count is invisible to JavaScript.

Layer iPhone 17 Pro Max example Source
CSS pixels 430 × 932 screen.width × screen.height
Rendered buffer 1290 × 2796 CSS px × DPR (3) — what the GPU draws
Physical panel 1320 × 2868 @ 460 PPI Hardware — not accessible via JS

Because devicePixelRatio is the CSS-to-rendered ratio (3), not the CSS-to-hardware ratio (~3.07), auto-detection reports the rendered resolution rather than the true hardware resolution. This causes a roughly 6 % error in pixel pitch, which in turn affects PPI, arc-minute thresholds, and mm character sizes.

Recommendation: select the matching device from the Preset list, or enter the hardware resolution and diagonal manually.
