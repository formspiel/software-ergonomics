Accessible SVG Experiment

Five variants of a paper plane icon exploring SVG accessibility techniques

How a screen reader announces an SVG depends on two things: how the file is embedded (<img>, inline, or <object>) and what accessibility markup is inside the SVG itself. This page shows five file variants with notes on what each technique does — and where it falls short.

1 — No accessibility markup

<!-- SVG contains no title or desc -->
<img src="paper-plane-optimised.svg" alt="A paper plane icon">

<!-- Decorative: suppress from screen readers entirely -->
<img src="paper-plane-optimised.svg" alt="">

When the SVG has no internal accessibility markup, the alt attribute on <img> is the sole source of the accessible name. Use a meaningful alt for informative icons and alt="" to hide decorative icons from screen readers. This is the recommended pattern for SVGs loaded via <img>.

2 — Inline <title> with ARIA

<svg role="img" aria-labelledby="plane-title" xmlns="..." viewBox="0 0 85 85">
  <title id="plane-title">A paper plane</title>
  ...
</svg>

When SVG is embedded inline, the <title> element provides the accessible name — but only if paired with role="img" and aria-labelledby pointing to the title's id. Without these ARIA attributes, screen reader support is inconsistent across browsers. When loading via <img>, the SVG <title> is not exposed as the accessible name; the alt attribute takes precedence.

3 — <title> inside an <img>

<!-- Inside the SVG -->
<title>This is a meaningful title</title>

<!-- In the HTML -->
<img src="paper-plane-with-title.svg" alt="A paper plane icon">

The SVG <title> is present in this file, but when loaded via <img> most browsers expose it only as a tooltip on hover — not as the screen reader accessible name. The alt attribute on <img> remains the accessible name. The SVG <title> is most useful for inline or <object> embeds (see section 2).

4 — Inline CSS with legacy high-contrast support

<style>
  .flyer { fill: #35A28F; }

  @media screen and (-ms-high-contrast: active) {
    .flyer { fill: windowText; } /* IE / old Edge only */
  }
</style>

Legacy technique. The -ms-high-contrast media query is a proprietary Microsoft extension supported only in IE and pre-Chromium Edge — neither of which is relevant today. The modern standard is @media (forced-colors: active) with CSS system colour keywords such as CanvasText. Inline <style> blocks inside SVG work in all current browsers, including when the SVG is loaded via <img>.

5 — External CSS via XML processing instruction

<?xml-stylesheet type="text/css" href="svg.css" ?>
<svg xmlns="..." viewBox="0 0 85 85">
  <title>This is a meaningful title</title>
  <g class="flyer"> ... </g> <!-- no inline fill -->
</svg>

The icon above renders without colour — an intentional demonstration of the technique's main limitation. The <?xml-stylesheet?> processing instruction loads an external CSS file, but only when the SVG is opened standalone in the browser or embedded via <object>. When loaded as <img>, the SVG is sandboxed: external resources are blocked, the stylesheet is never fetched, and .flyer has no inline fill fallback — leaving SVG's default black fill. Open the file directly to see svg.css applied correctly.
