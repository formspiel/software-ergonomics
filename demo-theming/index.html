How it works

This demo illustrates three layers of colour-scheme support, each building on the last:

  1. OS / browser preference — automatic, zero JS. The CSS color-scheme: light dark declaration tells the browser to follow the operating system setting. All colour values use the light-dark() function, which automatically picks the right value based on the active scheme.
  2. Forced colours — automatic, zero JS. Windows High Contrast Mode and similar OS accessibility settings are handled via a @media (forced-colors) block in the shared stylesheet, mapping all custom properties to system colours (Canvas, CanvasText, ActiveText).
  3. Explicit user choice — one line of JS. The Theme selector above overrides the OS preference by setting document.documentElement.style.colorScheme to "light" or "dark". This inline style takes precedence over the stylesheet, so light-dark() everywhere on the page resolves correctly — no custom properties or CSS overrides needed. Choosing Auto clears the inline style and hands control back to the OS. The choice is saved in localStorage and restored on the next visit.
