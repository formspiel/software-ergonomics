Prevent Tab Close

Browsers can warn users before they leave a page with unsaved data — protecting them from accidentally losing a half-written comment, a form, or a draft.

Try it out

  1. Open the comment form below and type something in any field.
  2. Try to close the tab or press the browser back button — the browser will ask you to confirm before leaving.
  3. Go back into the form and submit it. The warning is gone because the data was intentionally sent.
Open the comment form →

How it works

The warning is triggered by the browser's beforeunload event, which fires whenever the user tries to leave the current page — whether by closing the tab, pressing the back button, or navigating to a new URL.

Calling event.preventDefault() inside that listener tells the browser to show its native "Leave site?" dialog. The text and appearance of this dialog cannot be customised — browsers intentionally blocked that years ago to prevent phishing sites from mimicking system prompts.

A single boolean flag (hasUnsavedData) controls whether the warning fires. A single input listener on the <form> element tracks all fields automatically via event bubbling — no per-field wiring needed. On intentional submit, the flag is cleared so navigation proceeds without a warning.
